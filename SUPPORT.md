# Support

Thank you for using and contributing to Kepler!

If you need help, please use the appropriate channel below.

## Questions and General Help

For questions about using the project, setup, configuration, or general development help, please open a **Question / Help** issue using our GitHub issue template.

Before opening a new issue, please search existing issues to see if your question has already been answered.

## Bug Reports

If you have found a bug, please use the **Bug Report** issue template.

When reporting a bug, include:

- A clear description of the issue
- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Relevant environment details
- Screenshots or logs, if applicable

## Feature Requests

If you have an idea for a new feature or improvement, please use the **Feature Request** issue template.

Please describe the problem or use case and explain your proposed solution.

## Documentation

If you find an error or missing information in the documentation, please use the **Documentation Improvement** issue template.

## Security Issues

Please do **not** report security vulnerabilities through public GitHub issues.

For information about responsibly reporting security vulnerabilities, please refer to our [Security Policy](SECURITY.md).

## Before Asking for Help

Please make sure you have:

1. Read the project's documentation.
2. Checked existing GitHub issues and discussions.
3. Used the latest version of the project, if applicable.
4. Included relevant error messages, logs, or screenshots.
5. Provided enough information for others to reproduce the issue.

## Community Guidelines

Please be respectful and follow our [Code of Conduct](CODE_OF_CONDUCT.md) when interacting with maintainers and contributors.

Thank you for helping make Kepler a better project!
